Some Trouble

The Latex code is presented in Fig.1.1. In normal latex codes, double backslashes ( \\ ) is used to start a new line. However, due to the conflicts between the markdown renderer engine (marked.js) syntax and the latex syntax, it will not be rendered properly. "\\" will first be recognized as an escape character "\" by marked.js.

So, if you want to change line in a latex block, you need to type four backslashes (Fig.1.1). Some other syntax also share the same problems. Take 'Underscore' (_) as an example, in markdown, if a string was surrounded by two '_', the string will be in italics, while _ means subscript in Latex.
Trouble has been solved!

I changed the markdown rendering engine of my blog on Jun. 14, replaced the old one, marked.js, with markdown-it-plus. Now I don't need to type four backslashes in formulas anymore (see Fig.1.2), the "underscore problem" has also been solved. More importantly, the portability of the latex codes in my blogs is much better now 😄.
